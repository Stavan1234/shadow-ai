SHADOW AI
---

## 🎯 Project Goals

* **Jarvis-Style Companion**: Create a highly capable, super smart local AI desktop assistant inspired by Jarvis and the silent, invaluable helpers of *Shadow Slave*.


* **Absolute Local Sovereignty**: Provide a high level of user control with a system centered around mostly local, private thinking.


* **Consumer-Hardware Optimization**: Eliminate the requirement for high-end cloud instances or expensive dedicated GPUs, ensuring the tool runs smoothly on average laptops with 8GB to 16GB of RAM.


* **Structural Memory Foundation**: Utilize Cognee long-term knowledge graph layers extensively to achieve true context persistence across reboots.


* **Total Autonomy**: Achieve a fully offline, voice-first local system handling its own context routing and operating-system control workflows.



---

## 👁️ End Product Vision

* **The Two-Mind Memory Model**: A beautifully split cognitive architecture that manages memory across an instant, conversational "Conscious Mind" and a deep, permanent, relational "Subconscious Mind".


* **100% Offline Resilience**: A privacy-centric desktop application that retains its full operational power, context windows, and tool routing even with the laptop's WiFi turned completely off.


* **Voice-First Interface**: A fluid, microphone-and-speaker-driven user experience featuring zero-lag local audio transcription and highly expressive, realistic vocal synthesis.


* **Persistent Personal Identity**: An assistant that fundamentally recognizes who you are, tracks your core deadlines, and maintains specific situational context across independent runtime sessions.


* **Local System Execution Agent**: A tool that actively transforms conversational intents into physical actions within your laptop's environment, such as creating, moving, and reading directories or system files.

## 🛠️ Key Architectural Features

### 🧠 The Conscious Mind (Short-Term Layer)

* **RAM-Based Message Context**: Dynamically tracks and updates an in-memory Python list holding the last 10 conversational message exchanges.


* **Prompt History Injection**: Automatically injects these active session turns directly into every new LLM inference request as immediate conversational history.


* **Crash-Resilient File Mirroring**: Automatically flushes the active short-term list into a local `conscious_memory.json` file after every single turn to ensure current conversation logs survive sudden system reboots.



### 💾 The Subconscious Mind (Long-Term Layer)

* **Self-Hosted Knowledge Graph**: Utilizes Cognee to construct, map, and navigate an immutable, structured database of interrelated concepts on your local storage drive.


* **ECL Ingestion Pipeline**: Ingests new data through a specialized Extract, Cognify, and Load layout that isolates entities, classifies subjects, maps relationships, and commits edges directly to a local graph model.


* **Deterministic Fact Triggering**: Scans user inputs using an explicit, script-level keyword filter (`looks_like_fact`) to watch for phrases like *"my name is"*, *"my exam"*, *"deadline"*, and *"remember that"* before writing data to long-term memory.


* **Anti-Hallucination Safeguards**: Enforces strict prompt boundaries that tell the LLM to reply precisely from recorded graph fragments or state *"I don't have anything on record yet"* rather than inventing fictional facts.


* **Visual Topology Mapping**: Supports the export and generation of structural data charts showing the assistant's clear, multi-hop mental nodes (e.g., `[Stavan] → [has_exam] → [AWS Certification] → [scheduled_on] → [August 31]`).



### 📄 The Lazy Document Store

* **Sub-Graph Document Registry**: Processes complex multi-page files by reading text through `MarkItDown`, generating a concise 3-line summary via the LLM, and committing only the file path, name, and short summary to the graph.


* **On-Demand Context Retrieval**: Bypasses total document embedding ingestion; instead, when a user asks an explicit question about a document, Shadow pulls its registered file path, re-reads just that relevant section, and generates a response.



### 🖥️ Local Voice and Tool Infrastructure

* **Model Context Protocol (MCP)**: Utilizes open standard MCP routing to seamlessly plug system-level functional capabilities directly into the core Shadow orchestrator loop.


* **Automated File Commander**: Integrates `ellmos-filecommander-mcp` to expose 44 system-level tools capable of safely creating directories, listing files, and modifying your desktop workspace.


* **High-Speed Audio Capture**: Implements `faster-whisper` (large-v3 model optimized with INT8 CTranslate2 compression) to process speech-to-text processing 4× faster than standard Whisper networks.


* **ONNX Speech Generation**: Employs an ultra-lightweight, 82-million parameter `Kokoro TTS` engine running locally via ONNX to speak replies back to the user in a highly natural, completely offline voice.

## ⚡ Efficiency & System Optimization

### 📦 Hardware and Model Balancing

* **RAM-Tiered Scaling**: Dynamically pairs your physical laptop memory to optimized model weights, matching 8GB systems to the highly efficient `qwen3:7b` (~5 GB footprint) and 16GB tiers to `qwen2.5:14b` or `gemma4:12b`.


* **Quantization Headroom Allocation**: Targets precise 4-bit model variants (such as `qwen2.5:14b` running at Q4 quantization requiring ~9 GB of RAM) to guarantee enough leftover system RAM remains completely open for Windows and database operations.


* **Context Window Boundaries**: Explicitly locks down model parameters via options setting `num_ctx: 2048` and capping output blocks via `num_predict: 300` to prevent performance from lagging as the history grows.



### ⏱️ Latency and Execution Adjustments

* **Bypassing the Cache Trap**: Disables Cognee's internal session caching configurations to prevent an initialization bug that causes a 41-second query delay, routing short-term memory through raw local JSON logic instead.


* **Fire-and-Forget Asynchronous Writes**: Bypasses deep graph extraction during active conversation turns by using `self_improvement=False` and `run_in_background=True` inside `cognee.remember()`, ensuring long-term data saves never stall the immediate response.


* **Deferred Memory Consolidation**: Defers the computationally heavy 2-to-5 minute CPU graph-building process (`cognify()`) into single batched routines (`consolidate()`) executed only when you type it manually, close the application, or hit a 5-turn limit.


* **Endpoint Routing Isolation**: Isolates communication streams by keeping conversational chat requests directed to the primary `/v1` server endpoint, while forcing local vector embedding calls specifically to Ollama's native `/api/embed` path to avoid recurring 404 text errors.


* **Tokenizer Fallback Hardcoding**: Configures an environmental override for `HUGGINGFACE_TOKENIZER` pointing to local tokens like `bert-base-uncased` or `nomic-ai/nomic-embed-text-v1.5`, allowing local token counting to proceed seamlessly without crashing.


* **Permission Layer Bypassing**: Deactivates multi-user enterprise permission validation by injecting `ENABLE_BACKEND_ACCESS_CONTROL=false`, saving valuable processing loops and stopping runtime environment crashes.

## 📝 Technical System Details

* **Development Baseline**: Programmed for Windows 11 (10.0.26200) utilizing Python 3.13.5 and Ollama 0.30.11.


* **Isolated Virtual Environment**: Executes inside a completely self-contained local virtual env (`venv`) to ensure core libraries do not conflict with globally installed application packages.


* **Database Infrastructure**: Leverages file-based serverless local storage engines, running `ladybug` (Kuzu) for underlying graph data structures and `lancedb` for 768-dimension local vector calculations.


* **Core Dependencies**: Anchored tightly to a specific system manifest containing `cognee` (v1.2.2), `ollama`, `faster-whisper`, `kokoro-onnx`, `markitdown[ocr]`, `transformers`, `torch`, and `python-dotenv`.
