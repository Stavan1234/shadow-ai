### 🧠 1. The Memory Architecture: Beating the Database Locks

In the previous version, we discovered that file-based databases like Cognee's internal `ladybug` (for graphs) and `lancedb` (for vectors) will crash or throw locking errors if multiple tasks try to write to them at exactly the same time.

To fix this, we will implement a **Strict Sequential Async Queue**.

Instead of writing to the database the second you type a fact, S.H.A.D.O.W. will drop the fact into an internal waiting room (a queue). A single background worker (the "Subconscious Daemon") will silently process this queue one item at a time while you continue chatting seamlessly.

* **Conscious Mind:** Remains your blazing-fast local `conscious_memory.json` file.


* **Subconscious Write-Queue:** Uses Python's native `asyncio.Queue()` to completely eliminate database locking errors.
* **Dream Mode (Consolidation):** When you type `quit` or S.H.A.D.O.W. detects 10 minutes of idle time, it runs `cognee.cognify()` in one heavy batch to organize the graph, protecting your CPU during active work hours.



### ⚡ 2. The Model Footprint: Managing 16GB RAM

To give S.H.A.D.O.W. the ability to scrape the web, read OCR, and parse terminal outputs, the LLM needs strong logical reasoning.

* **The Brain:** Stick to **`qwen2.5:14b`** via Ollama. It requires roughly 9GB of RAM.


* **The Constraint:** This leaves you with ~7GB for Windows, your code editor, and system execution. You **must** keep the context window tightly capped at `num_ctx: 2048` or `4096`. If you let the context window balloon, Ollama will overflow into your CPU swap space, and your laptop will freeze.



### 🛠️ 3. The Action Layer: Model Context Protocol (MCP)

You absolutely do not need to write custom Python scripts for every single "wild" task. The secret to making S.H.A.D.O.W. endlessly flexible is the **Model Context Protocol (MCP)**.

Because MCP servers are heavily standardized (and frequently written in Node.js/TypeScript), you can instantly spin up isolated local servers that talk directly to S.H.A.D.O.W. via standard input/output (`stdio`).

Here is how you scale the system capabilities:

1. **File System & OS Control:** You run an open-source MCP file-system server. S.H.A.D.O.W. automatically inherits the ability to read directories, create folders, and modify files.
2. **Web Scraping & Research:** You plug in a Puppeteer/Playwright MCP server. If you ask about a new framework, S.H.A.D.O.W. uses the tool to silently open a headless browser, scrape the documentation, and summarize the new syntax.

async def memory_worker():
    """Runs continuously in the background, writing facts strictly one-by-one."""
    while True:
        fact = await memory_queue.get()

            print(f"[Subconscious Error] {e}")
